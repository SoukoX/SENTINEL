# Contributing to SENTINEL

Thanks for wanting to help! SENTINEL is a closed-source binary, but there are plenty of ways you can contribute.

---

## 🐛 Report Bugs

Open a [bug report](https://github.com/SoukoX/SENTINEL/issues/new?template=bug_report.md). Include:

- Your OS and version (e.g. `uname -a`)
- SENTINEL version (`sentinel --version` or the release tag)
- What you expected vs what happened
- Steps to reproduce
- Any logs or screenshots

---

## 💡 Suggest Features

Open a [feature request](https://github.com/SoukoX/SENTINEL/issues/new?template=feature_request.md). Tell us:

- What problem you're trying to solve
- How you imagine the feature working
- What tools or backends it could integrate with

---

## 📖 Improve Docs

Small fixes or big rewrites — both welcome.

- `README.md` — the main landing page
- `install.sh` — the installer script
- This file (`CONTRIBUTING.md`)

Submit a pull request directly. Every improvement counts.

---

## 🎨 UI/Frontend

The web interface (`ui.html`, `ui_phase2.js`) lives in this repo. If you know HTML/JS/CSS, you can:

- Improve the dashboard layout
- Add dark mode
- Make it mobile-friendly
- Add real-time progress visualization

Submit a PR with your changes.

---

## 🧪 Test & Report

Run SENTINEL against different targets and environments:

- Different Linux distros (Ubuntu, Arch, Fedora, etc.)
- macOS
- Docker containers
- Different tool combinations

Open issues for anything unexpected.

---

## 🌍 Spread the Word

This helps more than you think:

- ⭐ Star the repo on GitHub
- Share on Twitter/X, Reddit (r/netsec, r/cybersecurity, r/bugbounty), LinkedIn
- Write a blog post or make a video showing SENTINEL in action
- Add SENTINEL to Awesome Lists, tool collections, or security tool roundups
- Tell your pentester friends

---

## ☕ Sponsorship

If SENTINEL saves you time or helps your work, consider [sponsoring](https://github.com/sponsors/SoukoX). Funds go toward hosting, tool licenses, and development time.

---

## 🚀 Becoming a Core Tester

Active contributors who file quality bug reports and feature ideas may be invited to:

- Early access to new releases
- Direct input on roadmap priorities
- Private testing channel

---

## Guidelines

- Be respectful and constructive
- One issue per bug/feature — don't bundle unrelated things
- Search existing issues before opening a new one
- For questions, use Discussions (enable in repo Settings → Features) — not issues

---

Thanks again — every contribution, no matter how small, makes SENTINEL better.
